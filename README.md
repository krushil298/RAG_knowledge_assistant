# RAG Knowledge Assistant

A minimal **Retrieval-Augmented Generation (RAG)** assistant that answers questions
about your own documents — and *only* your documents. It retrieves the most relevant
passages from a local vector index and asks Gemini to answer strictly from that
context, so it cites its sources and refuses to make things up.

Built with **LangChain · Google Gemini · Chroma**.

## How it works

The pipeline runs in two phases:

**1. Ingest** (`ingest.py`) — run once, or whenever your documents change:

```
load .txt files  →  split into chunks  →  embed each chunk  →  store in Chroma
```

**2. Query** (`query.py`) — run any time to ask questions:

```
embed question  →  retrieve top-k chunks  →  grounded prompt  →  Gemini answers + shows sources
```

If the answer isn't in your documents, it replies:
`"I don't have enough information to answer that."`

## Setup

```bash
# 1. Create and activate a virtual environment
python3 -m venv .venv
source .venv/bin/activate

# 2. Install dependencies
pip install -r requirements.txt

# 3. Add your Gemini API key
cp .env.example .env
# then edit .env and paste your key (free at https://aistudio.google.com/apikey)
```

## Usage

```bash
# Build the vector index from the docs/ folder
python ingest.py

# Ask questions interactively
python query.py

# ...or ask a single question and exit (one-shot mode)
python query.py "What are the pre-screening steps?"
```

In interactive mode, type your questions at the `>` prompt; type `quit` to exit.

To use your own knowledge base, drop `.txt` files into `docs/` and re-run
`python ingest.py`.

## Project structure

```
ingest.py              # Phase 1: build the vector index
query.py               # Phase 2: ask questions
docs/                  # source documents (.txt)
requirements.txt       # Python dependencies
.env.example           # template for your API key
```

## Notes

- `.env`, `.venv/`, and `chroma_db/` are git-ignored — your API key never leaves
  your machine, and the index is regenerated by `ingest.py`.
